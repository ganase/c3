# DESIGN.md

## 目的

Codex / Claude Code のCLI作業を、楽天らしいブラウザUIとして扱えるようにする。

中心色は Rakuten Red `#BF0000`。

## 参照した考え方

`rakuten.design` の方向性を、このUIでは次の3点に置き換える。

- アクションを生み出す: 次に押すボタンがすぐ分かる
- 顧客体験を向上させる: CLIの難しさを画面で翻訳する
- ブランド価値を高める: 赤を強く使いながら、業務UIとして落ち着かせる

## カラーパレット

### Primary

- Rakuten Red: `#BF0000`
- Deep Crimson: `#8F0000`
- Light Crimson: `#FDECEC`

### Neutral

- Ink: `#1F1F1F`
- Text Subtle: `#666666`
- Border: `#D9D9D9`
- Surface: `#FFFFFF`
- Background: `#F7F7F7`

### State

- Success: `#0F7A3D`
- Warning: `#B45309`
- Info: `#1D4ED8`

## 色の使い方

- `#BF0000` は主要操作、選択状態、ブランドマークに使う
- 背景全体を赤くしない
- 赤い面は白文字を使う
- 長文本文に赤を使わない
- 注意やエラーは赤だけに頼らず、文言とアイコンも使う

## レイアウト

### 左端レール

VS CodeのActivity Barに近い入口。

- フォルダタブ
- 今後の拡張タブ

### 左ペイン

ルートフォルダとアプリ一覧。

- 新規作成
- 既存を開く
- ルート指定
- アプリフォルダ一覧
- 状態
- 履歴

### 中央ペイン

依頼と理解の確認。

- タイトル
- フォルダ名
- 指示入力
- 再利用ファイル
- 指示を実行
- 思考マップ
- 会話
- 作業ステップ

### 右ペイン

確認と結果。

- 要約
- プレビュー
- README.md

## UIトーン

- 説明は短くする
- 操作名は明確にする
- 見出しは強く、本文は控えめにする
- CLIログは作業ステップに変換する
- ユーザーの意図は実行前に図式化する

## コンポーネント

### Primary Button

- 背景: `#BF0000`
- 文字: `#FFFFFF`
- Hover: `#8F0000`
- 用途: 新規作成、指示を実行、許可

### Reusable Files

- 対象: `DESIGN.md`, `Agent.md`, `AGENTS.md`, その他Markdown
- 保存先: 新規作成時のフォルダ
- 表示: `ファイル名 → フォルダ名/ファイル名`
- 実行前に保存状態を表示する

### Folder Explorer

- ルートフォルダを指定できる
- ルート直下のフォルダを各アプリとして扱う
- フォルダ選択でセッション状態を更新する
- 選択中フォルダは赤い選択状態で示す

### Secondary Button

- 背景: `#FFFFFF`
- 枠線: `#D9D9D9`
- 文字: `#1F1F1F`
- 用途: 既存を開く、修正、タブ以外の補助操作

### Selected State

- 背景: `#FDECEC`
- 枠線: `#BF0000`
- 文字: `#8F0000`

### Step

- 完了: `#BF0000`
- 現在: `#FDECEC` + `#BF0000`
- 未実行: `#FFFFFF` + `#D9D9D9`

## 文言ルール

- 1文は短くする
- UI上で思想を語らない
- 詳細はREADME.mdへ逃がす
- 「確認」「実行」「許可」「記録」のような短い動詞を使う
- 一般ユーザーにCLI用語をそのまま出さない

## アクセシビリティ

- 赤地には白文字を使う
- 小さい赤文字だけで重要情報を伝えない
- ボタンは状態差が色だけにならないようにする
- フォーカス状態を明確にする
